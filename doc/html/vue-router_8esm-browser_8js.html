The only requirements forids are them to be unique on the page and to not be empty (id=""). NOTE: It's not clear to me if we should encode the + symbol in queries, it seems to be less flexible than not doing so and I can't find out the legacy systems requiring this for regular requests like text/html. In the standard, the encoding of the plus character is only mentioned for application/x-www-form-urlencoded (https://url.spec.whatwg.org/#urlencoded-parsing) and most browsers seems lo leave the plus character as is in queries. To be more flexible, we allow the plus character on the query, but it can also be manually encoded by the user.
